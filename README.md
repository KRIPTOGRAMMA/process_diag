# process_diag

Визуальная карта бизнес-процессов и архитектуры проекта: flowchart по каждому
домену, ER-диаграмма ключевых моделей, карта связей между доменами — всё в
одном самодостаточном HTML-файле (mermaid встроен, работает офлайн и внутри
Claude Code Artifacts).

Не автоматический конвертер: это шаблон + инструкция для LLM-агента, который
читает граф кодовой базы (построенный [graphify](https://github.com/safishamsi/graphify))
и сам решает, как сгруппировать процессы и что показать на диаграммах.

## Установка skill (для Claude Code)

Чтобы вызывать `/process-diag <путь>` в любом проекте, нужно один раз
скопировать skill из этого репозитория в личную папку skills:

```bash
mkdir -p ~/.claude/skills/process-diag
cp -r skills/process-diag/* ~/.claude/skills/process-diag/
```

Затем в файле `~/.claude/skills/process-diag/SKILL.md` заменить все
упоминания `<repo>` на реальный путь к этому клонированному репозиторию
(например `/home/you/Projects/process_diag`) — skill вызывает
`<repo>/build.sh` и читает `<repo>/INSTRUCTIONS.md`, поэтому путь должен
указывать туда, куда репозиторий реально склонирован на этой машине.

## Как пользоваться

### В Claude Code (после установки skill)

```
/graphify <путь-к-проекту>       # если графа ещё нет
/process-diag <путь-к-проекту>
```

Skill сам проверит граф, соберёт шаблон и заполнит его диаграммами.

### Вручную (без skill)

```bash
/graphify <путь-к-проекту>              # построить graph.json (если нет)
<repo>/build.sh <путь-к-проекту>
```

Скрипт соберёт `template.html` (с инлайн `mermaid.min.js`) в
`<путь-к-проекту>/<project>-processes.html` и напечатает инструкцию. Дальше
нужно попросить любого LLM-агента (Claude и т.п.) выполнить
`INSTRUCTIONS.md` — прочитать `graphify-out/graph.json` и `GRAPH_REPORT.md`,
и заполнить скопированный HTML диаграммами.

## Файлы

- `template.html` — шаблон страницы (стили, тёмная/светлая тема, сайдбар,
  инлайн mermaid.js) с плейсхолдерами `{{...}}` вместо контента.
- `mermaid.min.js` — локальная копия mermaid, встраивается в готовый HTML
  инлайн (так что итоговый файл работает без интернета и внутри CSP-строгих
  окружений вроде Claude Code Artifacts).
- `build.sh` — проверяет граф, собирает шаблон с инлайн `mermaid.min.js` в
  целевой `<project>-processes.html`, печатает инструкцию.
- `INSTRUCTIONS.md` — подробная инструкция для агента: как читать граф,
  какой mermaid-синтаксис использовать (включая важный нюанс с переносами
  строк внутри узлов), как заполнять плейсхолдеры.
- `skills/process-diag/SKILL.md` — Claude Code skill `/process-diag`,
  копируется в `~/.claude/skills/process-diag/` (см. "Установка skill" выше).
